## LAYER 6: CROSS-EPIC CONSISTENCY

### Warnings

1. **[WARNING] API error response format inconsistency (3 patterns)**
    - **Pattern A** `{ error, message }`: auth.ts, plugins/auth.ts, users.ts, versions.ts:721
    - **Pattern B** `{ code, message }`: All data/domain routes (correct pattern)
    - **Pattern C** `{ error }` (no message): system-config.ts:51,56
    - Frontend `apiClient` reads `body.code ?? 'UNKNOWN_ERROR'` — auth errors become 'UNKNOWN_ERROR'
    - Fix: Standardize all to Pattern B `{ code, message }`

2. **[WARNING] Frontend mutation hooks have no `onError` callbacks**
    - 14 `useMutation` hooks across use-enrollment.ts (4), use-revenue.ts (4), use-versions.ts (4), use-assumptions.ts (1), use-fiscal-periods.ts (1) — all have `onSuccess` but no `onError`
    - Some pages handle at call site (accounts, reference, assumptions), but enrollment and revenue pages silently swallow errors
    - Fix: Add `onError: (err) => toast.error(err.message)` to all mutations

3. **[WARNING] Logout uses raw fetch() with swallowed error**
    - File: `apps/web/src/stores/auth-store.ts:120-133`
    - Comment explains: `// Ignore logout failures` — intentional but inconsistent
    - Fix: Acceptable as-is (fire-and-forget), but add explicit comment explaining rationale

4. **[WARNING] API response field naming inconsistency (camelCase vs snake_case)**
    - camelCase: versions CRUD, enrollment, revenue, fiscal-periods, context, master-data
    - snake_case: auth (`access_token`, `expires_in`), users (`is_active`, `last_login_at`), audit (`user_id`, `table_name`), health (`uptime_seconds`), version compare (`total_revenue_ht`, `variance_absolute`)
    - Mixed within same domain: versions CRUD = camelCase, versions compare = snake_case
    - Fix: Standardize to camelCase (JS/TS convention). Highest priority: versions compare endpoint

5. **[WARNING] system-config validation errors use non-standard format**
    - Found: `{ error: string }` with no `code` or `message` field
    - File: `apps/api/src/routes/system-config.ts:51-52,56-58`
    - Fix: Change to `{ code: 'INVALID_CONFIG_VALUE', message: '...' }`

### Info

6. **[INFO] No React ErrorBoundary wrapping page components** — unhandled render errors crash entire app
7. **[INFO] Auth store login/refresh use raw fetch()** — intentional bootstrap pattern, correctly bypasses apiClient
8. **[INFO] Shared types (@budfin/types) used by frontend but not API** — API uses Zod schemas as source of truth. Low drift risk.
