================================================================================
KEY FINDINGS — CRITICAL RISKS
================================================================================

1. FLOATING-POINT PRECISION LOSS (SA-001)
   Risk Level: CRITICAL
   Impact: Revenue/cost line items diverge from Excel by 5-15 SAR
   Scale: 1,500 students × 50+ combinations × 12 months = 900,000+ operations
   Tolerance Exceeded: +/-1 SAR requirement violated in 15-25% of line items
   Root Cause: IEEE 754 doubles with multi-stage rounding
   Resolution: Use Decimal.js (JS) or decimal.Decimal (Python)
   Urgency: Must fix before architecture finalization

2. DATA MIGRATION INTEGRITY (SA-021)
   Risk Level: CRITICAL
   Impact: Silent import of malformed data creates subtle discrepancies
   Source: Known #REF! errors in source Excel
   Data Quality Issues: Formula errors, missing values, encoding corruption
   Risk if Unaddressed: Variance reports don't match Excel; audit failures
   Resolution: Pre-migration validation; block import on critical issues
   Urgency: Must implement before migration phase

3. ROUNDING ORDER DEPENDENCY (SA-002, SA-004, SA-006)
   Risk Level: HIGH
   Examples:
     - EoS provision diverges by ±840 SAR for 168 employees (±5 SAR each)
     - Discount cascade diverges by ±30 SAR per month (±0.05 SAR × 600 items)
     - YEARFRAC leap year boundary introduces ±5 SAR per employee
   Root Cause: Multiple rounding points; YEARFRAC implementation sensitivity
   Resolution: Document rounding sequence; maintain full precision internally
   Testing: Automated Excel baseline regression tests for all variants

4. DATA CONSISTENCY CASCADE FAILURES (SA-010, SA-013, SA-014)
   Risk Level: HIGH
   Scenarios:
     - Orphan revenue records after enrollment deletion
     - Fee grid cascades when grades are deleted
     - Version locking race conditions
   Impact: P&L includes ghost data; governance failures
   Resolution: Cascade delete with audit; optimistic locking with collision detection

5. CALCULATION EDGE CASES (SA-003, SA-005, SA-008, SA-009, SA-012)
   Risk Level: HIGH
   Issues:
     - Division-by-zero in zero-enrollment scenarios
     - CEILING boundary rounding causing utilization oscillation
     - Scenario parameter ambiguity (multiplicative vs. additive)
     - Non-tuition revenue weight validation missing
     - Negative enrollment in extreme scenarios
   Impact: Crashes, nonsensical calculations, silent errors
   Resolution: Validation, constraints, explicit documentation

6. AUDIT TRAIL DESIGN FLAWS (SA-025, SA-027)
   Risk Level: HIGH
   Problem: Bulk operations generate 840+ audit entries per single user action
   Scale: Cascade changes (enrollment → revenue → staff cost → P&L) multiply bloat
   Future State: 190K+ audit events over 7 years; queries timeout
   Impact: Audit trail becomes noise; impossible to find user actions
   Resolution: Batch audit with parent_id; aggregation views; cascade tracking
