================================================================================
KEY FINDINGS
================================================================================

1. DATA INTEGRITY RISKS
   - No duplicate detection for employee imports (PO-003 - CRITICAL)
   - Rounding logic not specified; fails Excel match requirement (PO-013 - CRITICAL)
   - Negative/out-of-range values not validated (PO-002, PO-016, PO-024)

2. VERSION MANAGEMENT GAPS
   - No mechanism to handle retroactive fee grid changes (PO-007 - CRITICAL)
   - Fee grids should be version-specific, not global (PO-007)
   - Variance analysis doesn't attribute enrollment vs. assumption changes (PO-008)
   - Version unlock workflow not specified (PO-021)

3. BUSINESS RULE AMBIGUITIES
   - Mid-year employee joining not covered; assumes Sep start (PO-004)
   - Salary augmentation assumed global in Sep; doesn't support staggered dates (PO-031)
   - GOSI/Ajeer transition for nationality changes not addressed (PO-006, PO-019)
   - Scenario parameter sequencing (fee collection vs. scholarships) unclear (PO-009)
   - Plafond treated as soft target; no validation if exceeded (PO-014)

4. ACADEMIC CALENDAR ISSUES
   - AY1 vs. AY2 fee grids need per-period enrollment detail (PO-011)
   - Zero-enrollment grades not handled explicitly (PO-001)
   - Mid-year enrollment changes trigger no staffing workflow (PO-005)
   - HSA Jul-Aug exclusion not clearly specified (PO-018)

5. REPORTING BOUNDARIES
   - Zero enrollment, zero revenue scenarios allowed but not flagged (PO-025)
   - Monthly revenue sum != annual (rounding) not addressed (PO-026)
   - Deleted employees not soft-deleted; version comparison breaks (PO-033)
   - Capacity over-utilization doesn't trigger staffing adjustment (PO-014)

6. MISSING SPECIFICATIONS
   - Revenue recognition policy (cash vs. IFRS accrual) not defined (PO-030 - CRITICAL)
   - Currency handling (SAR-only vs. multi-currency) not specified (PO-022)
   - Audit trail retention lifecycle (delete/archive after 7 years) not defined (PO-027)
   - Role change impact on user permissions not specified (PO-028)

================================================================================
IMPACT ON USER WORKFLOWS
================================================================================

CAO / Chief Accounting Officer:
- Cannot safely unlock & modify locked versions (PO-021 blocks workflow)
- Cannot compare versions with enrollment deltas (PO-008 lacks variance attribution)
- Cannot handle mid-year hiring adjustments (PO-004 assumes Sep start)
- Faces rounding discrepancies vs. Excel (PO-013 blocks Excel comparison)

Finance Manager:
- Duplicate employee imports will corrupt payroll (PO-003 breaks calculation)
- Negative discounts could create -ve revenue (PO-002, PO-024)
- Fee grid retroactive changes not supported (PO-007 blocks correction scenario)
- Mid-year enrollment changes cause staffing warnings but no workflow (PO-005)

Department Heads (HR/Staffing):
- Cannot stagger salary augmentation dates (PO-031 assumes global Sep)
- Cannot update employee joining date retroactively (PO-012 blocks HR correction)
- Cannot track nationality transitions for GOSI/Ajeer (PO-006, PO-019)

Finance Team (Monthly Close):
- Monthly P&L may not reconcile to annual if rounding is wrong (PO-026)
- Revenue recognition policy undefined; could affect IFRS reporting (PO-030)
- EoS retroactive adjustments not supported (PO-012 blocks correction)
