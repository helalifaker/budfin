# Appendix A — Functional Requirement Traceability Matrix

This appendix maps every functional requirement from PRD v2.0 to the TDD design sections, responsible components, implementing API endpoints, relevant database tables, and test types. The matrix ensures complete traceability from business requirements through technical design to verification.

**Legend:**

- **TDD File:** Which `docs/tdd/` file contains the relevant design (by file number)
- **Component:** Which component from Section 4 (`02_component_design.md`) implements the requirement
- **API Endpoints:** Primary REST endpoints involved
- **DB Tables:** Primary database tables involved
- **Tests:** U = Unit, I = Integration, R = Regression, E2E = End-to-End

**FR Source Key:**

- **PRD-defined:** FR IDs that map directly to a numbered requirement in PRD v2.0 §8 (FR-VER, FR-ENR, FR-CAP, FR-REV, FR-DHG, FR-STC, FR-PNL, FR-SCN, FR-MDM, FR-AUD, FR-DSH, FR-INP)
- **TDD-internal:** FR IDs generated by TDD for system infrastructure not explicitly enumerated in PRD §8 but derived from PRD requirements (FR-SEC, FR-CTX, FR-EXP, FR-IMP, ACT). These are now formally added to PRD §8 — see Change Log below.
